# PhotoRank PWA — Frontend

Mobile-first React PWA for PhotoRank. Built with Vite. No UI framework — the
design tokens are hand-extracted from the approved Claude Design prototype
(archived in [`_design/`](_design/)).

## Run

```bash
cd frontend
npm install
npm run dev        # http://localhost:5173
npm run build      # → dist/
npm run preview    # serve the production build (PWA + service worker active)
```

The service worker only registers in production builds, so use `npm run preview`
(not `dev`) to exercise offline behaviour and Add-to-Home-Screen.

## API

Talks to `POST /rank` at `https://photorank.job-joseph.com` (SPECS §6).
Override for local backend testing:

```bash
VITE_API_BASE=http://localhost:8007 npm run dev
```

`mode` is intentionally **not** sent — the server auto-detects burst vs set
from EXIF timestamps.

## Structure

```
src/
├── main.jsx              # mount + service-worker registration
├── App.jsx               # state machine: upload → loading → results → error
├── styles.css            # all design tokens + component CSS
├── config.js             # API base, profiles, axis labels, file limits
├── api.js                # rankPhotos() — multipart POST + friendly error mapping
├── compress.js           # client-side downscale to ~1.5 MP (HEIC → JPEG)
├── usePwaInstall.js       # Add-to-Home-Screen (beforeinstallprompt + iOS hint)
├── lib/breakdown.js      # normalize score_breakdown → bar geometry
├── screens/              # UploadScreen, LoadingScreen, ResultsScreen
└── components/           # HeroCard, RankCard, AxisBar, MessageScreen
public/
├── manifest.webmanifest
├── sw.js                 # offline app shell — never caches /rank
└── icons/                # generated by scripts/gen-icons.mjs
```

## Key behaviours

- **Score breakdown** renders dynamically from whatever axes the API returns
  (6 in set mode, 4 face/full-image axes in burst mode). Bar width = the axis's
  contribution; the dashed cap = the axis's max (`weight × 10`); each row shows
  the explicit `raw × weight = contribution` math.
- **Real previews:** uploaded photos are compressed client-side and the same
  blob is both uploaded and shown as the hero/thumbnails, mapped back by
  `photo_id`. If a browser can't decode the file (e.g. HEIC outside iOS Safari)
  it falls back to the gradient placeholder and uploads the original.
- **Privacy:** preview object URLs are revoked on reset; nothing is persisted;
  the service worker never caches the scoring API.
- **Offline:** the upload shell loads offline and shows a "connect to score"
  banner; attempting to rank offline shows a graceful message, never a raw error.
